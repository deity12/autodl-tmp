# Codex 提示词使用指南

## 📌 应该使用哪个提示词文件？

### **推荐：`CODEX_QUICK_PROMPT.txt`** ⭐

**为什么选择这个？**
- ✅ **简洁明了**：包含所有核心要点，不会太长
- ✅ **直接可用**：可以直接复制给 Codex，无需修改
- ✅ **重点突出**：强调直接点击运行、日志系统、配置写在文件里
- ✅ **适合快速开始**：Codex 可以快速理解需求

**如何使用**：
1. 打开 `CODEX_QUICK_PROMPT.txt`
2. 复制全部内容
3. 粘贴给 Codex
4. 说："请根据以上提示词优化我的 Graph-RWKV 项目代码"

---

### **详细版：`CODEX_OPTIMIZATION_PROMPT.md`**

**什么时候使用这个？**
- 📖 需要详细的代码示例时
- 📖 需要深入理解优化方向时
- 📖 需要参考完整的实现模板时

**如何使用**：
1. 先使用 `CODEX_QUICK_PROMPT.txt` 给 Codex
2. 如果 Codex 需要更多细节，再参考这个文件
3. 或者直接复制相关章节给 Codex

---

## 🎯 核心要求总结（给 Codex 的快速提示）

如果只想给 Codex 一个简短提示，可以这样说：

```
请根据以下要求优化我的 Graph-RWKV 项目代码：

1. 重构为 5 个独立可运行脚本（可以直接点击运行，无需命令行）：
   - 1_preprocess_data.py
   - 2_build_graph.py
   - 3_train.py（全量模型）
   - 3_train_ablation.py（消融实验）
   - 4_evaluate.py

2. 配置直接写在 py 文件里（不使用 config.yaml，已删除），所有配置参数在脚本顶部用大写常量定义

3. 每次运行自动保存日志到文件（包含时间戳），日志文件命名：{script_name}_{YYYYMMDD}_{HHMMSS}.log

4. 充分利用 48GB GPU（batch_size=1024+）和 90GB RAM（数据预加载）

5. 参考论文源码：RWKV-LM, ADGAT, PyTorch Geometric
```

## 📋 使用流程建议

### **步骤 1：准备提示词**
```bash
# 复制快速提示词
cat CODEX_QUICK_PROMPT.txt
# 或者复制完整提示词
cat CODEX_OPTIMIZATION_PROMPT.md
```

### **步骤 2：给 Codex 指令**
```
请根据以上提示词优化我的 Graph-RWKV 项目代码。

优先完成：
1. 脚本重构（5个独立可运行脚本）
2. 日志系统（每次运行自动保存）
3. 配置写在文件里（不使用 config.yaml）

然后进行性能优化。
```

### **步骤 3：检查结果**
- ✅ 脚本可以直接点击运行
- ✅ 配置参数在脚本顶部
- ✅ 日志自动保存到文件
- ✅ 日志文件包含时间戳
