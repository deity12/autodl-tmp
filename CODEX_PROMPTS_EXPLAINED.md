# Codex 提示词文件说明

## 📁 三个文件的作用和区别

### 1. **CODEX_QUICK_PROMPT.txt** ⭐（推荐使用）

**文件大小**：195 行  
**用途**：快速提示词，直接给 Codex 使用  
**特点**：
- ✅ **简洁明了**：包含所有核心要点，不会太长
- ✅ **直接可用**：可以直接复制给 Codex，无需修改
- ✅ **重点突出**：强调直接点击运行、日志系统、配置写在文件里
- ✅ **适合快速开始**：Codex 可以快速理解需求

**使用场景**：
- 第一次给 Codex 优化代码时使用
- 需要快速开始优化时使用
- 作为主要提示词文件

**使用方法**：
```bash
# 1. 打开文件
cat CODEX_QUICK_PROMPT.txt

# 2. 复制全部内容

# 3. 粘贴给 Codex，然后说：
"请根据以上提示词优化我的 Graph-RWKV 项目代码"
```

---

### 2. **CODEX_OPTIMIZATION_PROMPT.md**（详细参考）

**文件大小**：1633 行  
**用途**：详细的优化提示词，包含完整代码示例  
**特点**：
- 📖 **内容详细**：包含完整的代码示例和实现模板
- 📖 **深入参考**：详细说明每个优化方向
- 📖 **代码模板**：提供可直接使用的代码模板
- 📖 **适合深入学习**：理解优化思路和实现细节

**使用场景**：
- Codex 需要更多细节时参考
- 需要查看具体代码实现时
- 需要深入理解优化方向时
- 作为参考文档，不直接复制给 Codex

**使用方法**：
```bash
# 1. 先给 Codex 快速提示词
# 2. 如果 Codex 需要更多细节，再参考这个文件
# 3. 或者复制相关章节给 Codex
```

---

### 3. **README_CODEX_PROMPT.md**（使用指南）

**文件大小**：101 行  
**用途**：说明如何使用提示词文件的指南  
**特点**：
- 📋 **使用说明**：解释如何使用提示词文件
- 📋 **文件对比**：对比不同提示词文件的区别
- 📋 **使用流程**：提供使用流程建议
- 📋 **快速参考**：给 Codex 的简短提示模板

**使用场景**：
- 第一次使用时阅读
- 了解提示词文件的区别
- 快速查找使用方法
- **不直接给 Codex**，是给人看的指南

**使用方法**：
```bash
# 阅读这个文件，了解如何使用提示词
cat README_CODEX_PROMPT.md
```

## ❓ 常见问题

### Q: 我应该用哪个文件？
**A**: 推荐使用 `CODEX_QUICK_PROMPT.txt`，它包含所有核心要点，适合直接给 Codex。

### Q: 三个文件都要用吗？
**A**: 不是。`README_CODEX_PROMPT.md` 是给人看的指南，`CODEX_QUICK_PROMPT.txt` 是主要给 Codex 的，`CODEX_OPTIMIZATION_PROMPT.md` 是详细参考。

### Q: 可以合并成一个文件吗？
**A**: 可以，但分开更好：
- 快速提示词：简洁，Codex 容易理解
- 详细参考：需要时才看，不会让提示词太长
- 使用指南：帮助用户理解如何使用
